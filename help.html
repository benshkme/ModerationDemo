1 Overview

The Kaltura Moderation Demo is a browser-based tool that demonstrates Kaltura's Reach content moderation capabilities. It connects directly to the Kaltura API using a Kaltura Session (KS) and provides three main areas:

  • Entry Vendor Tasks — lists all moderation jobs for your account, shows their status and compliance result, and lets you order new jobs.
  • Review — shows a detailed moderation report (violations grouped by rule, confidence scores, timestamps) alongside the Kaltura Player for the analysed entry.
  • Policies — placeholder for future CRUD management of moderation policies.
ℹ️
No server, database, or build step is required. The app is pure HTML + JavaScript and calls the Kaltura API directly from the browser.

Running locally

Because the app uses fetch() to call the Kaltura API, it must be served over HTTP (not opened directly as a file:// URL) to avoid CORS issues.

  1. Clone or download the repository:
    git clone https://github.com/benshkme/ModerationDemo
  2. Navigate to the project folder:
    cd ModerationDemo
  3. Start a local HTTP server. With Python 3:
    python3 -m http.server 8080
  4. Open http://localhost:8080 in your browser.
💡
Any static file server works — VS Code's Live Server extension, npx serve ., npx http-server, Nginx, or Apache.

3 Connecting to Kaltura

All Kaltura API calls are authenticated with a Kaltura Session (KS). The connection bar at the top of every page contains three inputs:

FieldRequiredDescription
Kaltura Session (KS) Required A signed session token from your Kaltura account. Generate one in the KMC under Settings → Integration Settings → Generate KS, or via the API using session.start. The field is masked for security.
Service URL Optional Base URL of your Kaltura deployment. Defaults to https://www.kaltura.com. Change this for on-premises or regional SaaS deployments (e.g. https://www.eu.kaltura.com).
Player uiconf ID Optional* The numeric ID of a Kaltura Player v7 configuration from your account. Required for the Review tab player. Find it in the KMC under Studio → Universal Studio.

Obtaining a Kaltura Session

  1. Log in to the Kaltura Management Console (KMC).
  2. Go to Settings → Integration Settings.
  3. Click Generate KS and select an expiry time and privileges.
  4. Copy the generated token and paste it into the KS field.
⚠️
A KS is a sensitive credential. Treat it like a password — do not share it or commit it to version control. It expires after the configured TTL (default: 24 hours).

Player uiconf ID

If you don't enter a uiconf ID before opening the Review tab, the player area will show an inline prompt where you can enter it. The value is saved in sessionStorage so it persists across page refreshes within the same browser tab.

After clicking Connect, the app calls session.get to validate the KS and detect your Partner ID automatically. The status badge in the header turns green when connected.

The Entry Vendor Tasks tab lists all KalturaEntryVendorTask records for your account. Each record represents one moderation (or other Reach) job submitted for a specific entry.

Ordering a new moderation job

Click the + Order Job button in the toolbar to open the order form.

  1. Catalog Item — shown only if your account has multiple moderation catalog items (KalturaVendorModerationCatalogItem). If there is only one, it is selected automatically.
  2. Entry — browse or search the list of video entries from your account. Type a name or entry ID in the search box and press Enter or click Search. Click a row to select the entry.
  3. Reach Profile — select the reach profile (policy configuration) to apply. Loaded automatically from reachProfile.list.
  4. Policy ID — enter the numeric ID of the moderation policy to run. This is set as taskJobData.policyIds on the submitted task (KalturaModerationVendorTaskData).
  5. Click Submit Order. On success the modal closes and the task list refreshes automatically.
ℹ️
Ordering a job requires your account to have at least one active moderation catalog item (vendorCatalogItem.list with serviceFeatureEqual=15). Contact your Kaltura account team if none are available.

5 Review

The Review tab opens when you click a Review button on a Ready moderation task. It provides a split view: the moderation report on the left and the Kaltura Player on the right.

Moderation results panel

The left panel shows three sections:

SectionDescription
Job Details Task ID, Entry ID (with KMC link), status, service feature, vendor partner, and creation date.
Moderation Results

Parsed from taskJobData.moderationOutputJson. Shows the policy number, total violation count, and a Complies / Does Not Comply badge.

Violations are grouped by rule. Each rule is a collapsible row (collapsed by default) — click to expand and see every individual violation with its reason label and timestamp button.

A coloured dot indicates severity: 🔴 high (≥ 80), 🟡 medium (≥ 60), 🟢 low.

Summary Per-policy score and compliance result from summary[policyId].comply.

Video player

The right panel embeds the Kaltura Player v7 using the KalturaPlayer.setup() JavaScript API (not an iframe). This gives direct programmatic access for timestamp seeking. The player loads automatically with autoplay enabled when a review opens.

If no uiconf ID was provided in the header, an inline input appears inside the player area — enter the ID there and click Load Player.

Timestamp links

Every violation row in an expanded rule section shows a timestamp button (e.g. 01:23). Clicking it calls player.currentTime = seconds; player.play() — the player immediately seeks to that position and starts playing, so you can watch the flagged scene in context.

💡
The raw moderationOutputJson is always available at the bottom of the results panel as a collapsible Raw JSON section — useful for debugging or verifying the exact vendor output.

7 Key concepts

Kaltura Reach

Kaltura Reach is a managed services platform that connects Kaltura with third-party AI/ML vendors for automated media enrichment — including transcription, translation, and content moderation. Jobs are represented as KalturaEntryVendorTask objects.

Vendor Catalog Item (KalturaVendorModerationCatalogItem)

A catalog item defines a specific vendor service offering — for example "AI moderation by vendor X". It has a serviceFeature (e.g. 15 = MODERATION) and links to a vendor partner. You can only order jobs for catalog items that are active on your account.

Reach Profile (KalturaReachProfile)

A reach profile is an account-level configuration that defines settings such as default vendors, credit allocation, and language preferences. When ordering a job, you select a reach profile to govern how the task is processed. Reach profiles are different from moderation policies.

Policy ID

A moderation policy defines the ruleset applied during content analysis (which categories to detect, severity thresholds, etc.). Policy IDs are set on the task via taskJobData.policyIds using KalturaModerationVendorTaskData. The results in moderationOutputJson.summary are keyed by policy ID.

Moderation Output JSON

When a moderation job completes, the vendor report is stored in entryVendorTask.taskJobData.moderationOutputJson. Its top-level structure is:

FieldDescription
violations[]Array of individual detections. Each has id (rule ID), rule (description), reason, severity (0–100), and start_time (seconds).
summaryObject keyed by policy ID. Each value is { score: number, comply: boolean }. comply: false means the content violates the policy.

8 Troubleshooting

SymptomLikely cause & fix
Connection fails / "KalturaAPIException" The KS may be expired or have insufficient privileges. Generate a fresh KS with admin-level access.
Task list is empty The time-window filter defaults to Last 24 hours. Widen it to Last 30 days or All time. Also check that the Service Feature filter matches your jobs.
"No moderation catalog items found" Your account has no active moderation catalog items (vendorCatalogItem.list with serviceFeatureEqual=15 returns empty). Contact your Kaltura account team to have Reach Moderation enabled.
Player does not appear A Player uiconf ID is required. Enter it in the header bar or in the inline prompt inside the player area on the Review tab. Ensure the uiconf ID is a Player v7 (PlayKit) configuration, not a legacy kWidget player.
Timestamp seek does not work Ensure the player has fully loaded before clicking a timestamp. If the player shows a loading spinner, wait for it to finish buffering.
Moderation results show "—" (no data) The task must have status Ready and a non-empty taskJobData.moderationOutputJson. If the task completed but the field is empty, the vendor may not have returned a report — check with your Kaltura account team.
CORS errors in browser console The app must be served via HTTP (not opened as a file:// URL). Run a local server as described in the Installation section.
ℹ️
Open your browser's developer console (F12) for detailed API error messages. All Kaltura API errors include a code and message field that pinpoints the issue.
